## Purpose

This checklist verifies that every resident's environment is functional before Week 1 begins. Do not proceed with content delivery until all items are checked. A resident with a broken environment will fall behind permanently.

---

## Individual Resident Verification

**Resident Name:** _________________ **Date:** _________ **Verified By:** _________________

### R Installation

- [ ] Can open RStudio (or R from terminal)
- [ ] Can execute `R --version` in terminal and reports 4.3.0 or newer
- [ ] Can install a package: `install.packages("tidyverse")`
- [ ] Package loads without error: `library(tidyverse)`

### Git Installation

- [ ] Can open Terminal (macOS/Linux) or Git Bash (Windows)
- [ ] Can execute `git --version` and reports 2.40.0 or newer
- [ ] Can execute `git config --global user.name` and see their name
- [ ] Can execute `git config --global user.email` and see their email

### GitHub Connectivity

- [ ] Has GitHub account with 2FA enabled
- [ ] Can execute `ssh -T git@github.com` and see "Hi username! You've successfully authenticated"
- [ ] Can clone a repository: `git clone git@github.com:balinesthesia/mds-curriculum.git`
- [ ] Can navigate into cloned directory and see files

### Quarto Installation

- [ ] Can execute `quarto --version` and reports 1.4.0 or newer
- [ ] Can render a basic document (create `test.qmd` with one heading, render to HTML)
- [ ] HTML output opens in browser without errors

### File System Navigation

- [ ] Can open terminal in a specific directory
- [ ] Can navigate up/down directory tree (`cd`, `cd ..`, `ls`/`dir`)
- [ ] Can create a new directory: `mkdir test-folder`
- [ ] Can create a new file: `touch test.txt` (macOS/Linux) or `type nul > test.txt` (Windows CMD)
- [ ] Can move and rename files

1. **Do not start Week 1**
